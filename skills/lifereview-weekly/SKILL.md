# lifereview-weekly — 每周回顾

每周生成一次完整周报，覆盖 7 天全部维度。

## 运行

```bash
python3 ~/.life-log/tmp/weekly-review.py
```

周报输出到 `~/.life-log/2026-week-XX.md`

## 周报结构

1. **一周主题**：1-2 句概括
2. **逐日快照表**：地点/RHR/HRV/睡眠/步数/会议/Looki/关键词
3. **日历分析**：每天会议数+时长+亮点，日类型分类（工作日/差旅/假期）
4. **生理趋势图**：ASCII chart — RHR、HRV、睡眠、步数
5. **运动与恢复**：每天运动类型+活跃能量
6. **🆘🍬 信号汇总**：7 天求救/代偿/恢复矩阵
7. **已验证规律**：本周新证据
8. **待验证假设**
9. **本周值得注意**

## 数据源

### 全部参考 lifereview-daily 的数据源配置

关键文件：
- `~/.life-log/tmp/health-extract.py` (v4) — 逐个日期提取
- `~/.life-log/tmp/daily-narrative.py` — 叙事引擎
- Looki API — Python 直连 IP（见 daily skill）
- 飞书日历 — lark-cli calendar +agenda --start DATE --end DATE

### 周报日历数据拉取

对周内每一天逐个调用 `lark-cli calendar +agenda`，应用三层漏斗（见 daily skill）。

排除全天型「公共假期中」标记（00:00-23:59 的全天事件不计入小时统计）。

## 关键规则

### HAE 数据聚合 ⚠️
- 步数、活跃能量等必须用 **日总和**（sum of hourly），不是首个条目
- 已修复：health-extract.py v4 中 steps 和 active_energy 改为 sum()
- 之前因取首个小时条目导致步数被低估约 100 倍
- 接入任何新指标前必须确认其 HAE 存储粒度——见 daily skill 的数据自查清单

### 生理数据解读
- HRV 对比只用夜间窗口中位数（0:00-9:00），不包含白天数据
- RHR 使用中位数（抗离群）
- 所有生理指标解读限于数据形态标记，不做医学诊断
- VO2Max 需户外运动触发 Apple Watch 测算，无户外运动时不更新

### 物质摄入标记
- 仅标记用户确认或 Looki 明确捕捉的事件
- 不自行推断物质使用

### 日历参与规则
- 同 daily skill 的三层漏斗
- 全天型标记事件（00:00-23:59）不计入日时长统计
- 假期日标注：自创行程（飞/跑步/去机场）仍记录但不计为正式会议

### 运动判断
- 步数 + 活跃能量 + 日历中的运动类日程 = 运动日判定
- 运动日 RHR 可能升高——这可能是运动消耗而非异常
- 周运动频率底线：3-4 次/周（6 年数据验证）

## 四大防护规则 ⚠️

**周报生成必须遵守 daily skill 中的全部规则：**

1. **Looki API 用 Python 直连 IP**（禁止 curl — VPN TLS SNI 拦截）
2. **生成前必须跑 health-freshness.py** 检查每 7 天数据的新鲜度
3. **消息发送用 stdin**（禁止 argv — shell 会破坏换行符）
4. **所有 HAE 指标按 sum()/median()/first 分类聚合**（禁止假设「取第一条就行」）
5. **运动数据优先查 Workouts/ 目录**（独立管线，比主 JSON 快 7h+）
6. **文件夹 mtime ≠ 文件 mtime**（iCloud 可能只更新目录元数据）

## 周报格式 ⚠️ v2.3 — 借鉴 Mira 裁决格式

周报不以"逐日描述"开头，而以**裁决 + 证伪**开头。让用户 10 秒内知道这周的关键信号。

### 结构

1. **一句话裁决**（必选）——红黄绿灯 + 一句话核心判断
2. **为什么**（必选）——3-5 条证伪推理，每条 = "观察到什么 + 先别怪什么 + 更可能是什么"
   - 深睡连跌 → "先别怪睡眠时长（时长正常），是结构出了问题"
   - RHR 抬升但无持续趋势 → "不是过度训练，更像单次高强度后的正常恢复周期"
3. **下周三个决策**（必选）——训练/饮食/工作，每条带具体动作和触发条件
4. **本周趋势**（必选）——3-5 条滚动趋势，每条一句话
5. **参考表**（必选）——7 天 × 关键指标矩阵，底部一行标注个人基线和阈值
6. **数据审计**（必选）——每源一行状态 + 欠采标注 + 异常点说明
7. **逐日快照**（可选·保留）——原来的表格，放在参考表之后
8. **每周新视角**（可选·保留）——从 15 视角中选一个做周维度的深度分析

### 与日报的区别

- 日报是"今天能做什么"，周报是"这周的趋势说明了什么"
- 日报的证伪针对单日异常，周报的证伪针对跨日趋势
- 日报的决策是即时建议，周报的决策是下周的杠杆点
