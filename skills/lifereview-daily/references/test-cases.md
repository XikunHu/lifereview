# 测试用例 — 已知失败模式

每次修改 Skill 或脚本后，逐条检查以下场景。

## 1. 数据聚合

| 测试项 | 预期 | 曾失败 |
|--------|------|--------|
| 步数取 sum() | 全天汇总，非首个条目 | 6/20 显示 8 步（实际 13422） |
| 活跃能量取 sum() | 全天汇总 | 同上 |
| RHR 取 median() | 中位数抗离群 | — |
| HRV 取夜间 median() | 0-9点窗口 | 白天数据会污染恢复评估 |
| 新增指标前确认粒度 | sum/median/first 分类 | — |

## 2. 数据新鲜度

| 测试项 | 预期 | 曾失败 |
|--------|------|--------|
| 每次报告前跑 health-freshness.py | 实时检查，不缓存 | 6/24 用16:00缓存，漏掉21:29更新的篮球数据 |
| mtime_age_h >4h 标警告 | ⚠️ 过期标注 | — |
| mtime_age_h >8h 标严重过期 | 🔴 降低HAE权重 | — |
| 看文件mtime而非文件夹mtime | 准确判断 | iCloud 可能只更新目录元数据 |

## 3. Looki 数据

| 测试项 | 预期 | 曾失败 |
|--------|------|--------|
| 调 API 先 realtime 后 moments | 两条管道独立 | — |
| Moments=0 时查 realtime | 不误判离线 | 6/24 moments=0 但 realtime 有数据 |
| Python 调用禁止 curl | 绕过 VPN SNI 拦截 | 6/23 curl 返回空 |

## 4. 日历过滤

| 测试项 | 预期 | 曾失败 |
|--------|------|--------|
| 只看 self_rsvp_status=accept | 过滤未接受 | — |
| 排除系统日程 | Proma/Looki 不计入 | — |
| 排除提醒型 | 交费/还款/挂号等不计入 | — |
| 不排除真实耗时 | 洗/修/打扫/审阅/打电话计入 | — |
| API 失败时保留已有数据 | 不覆盖为空 | — |

## 5. 消息发送

| 测试项 | 预期 | 曾失败 |
|--------|------|--------|
| 消息体走 stdin | 换行符正常 | argv 会把 \n 当字面量 |
| 飞书 chat_id 正确 | 消息送达 | — |

## 6. 日期时态

| 测试项 | 预期 | 曾失败 |
|--------|------|--------|
| 用「当天」非「今天」 | 时态正确 | "今天"会造成时态穿越 |
| 睡眠读 X+1 日文件 | 归属正确 | 睡眠记在醒来日 |
