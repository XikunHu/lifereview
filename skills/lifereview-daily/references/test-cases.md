# 测试用例 — 已知失败模式

每次修改 Skill 或脚本后，逐条检查以下场景。

## 1. 数据聚合

| 测试项 | 预期 | 曾失败 |
|--------|------|--------|
| 步数取 sum() | 全天汇总，非首个条目 | 6/20 显示 8 步（实际 13422） |
| 活跃能量取 sum() | 全天汇总 | 同上 |
| RHR 取 median() | 中位数抗离群 | — |
| HRV 取夜间 median() | 0-9点窗口 | 白天数据会污染恢复评估 |
| 新增指标前确认粒度 | sum/median/first 分类 | — |

## 2. 数据新鲜度

| 测试项 | 预期 | 曾失败 |
|--------|------|--------|
| 每次报告前跑 health-freshness.py | 实时检查，不缓存 | 6/24 用16:00缓存，漏掉21:29更新的篮球数据 |
| mtime_age_h >4h 标警告 | ⚠️ 过期标注 | — |
| mtime_age_h >8h 标严重过期 | 🔴 降低HAE权重 | — |
| 看文件mtime而非文件夹mtime | 准确判断 | iCloud 可能只更新目录元数据 |

## 3. Looki 数据

| 测试项 | 预期 | 曾失败 |
|--------|------|--------|
| 调 API 先 realtime 后 moments | 两条管道独立 | — |
| Moments=0 时查 realtime | 不误判离线 | 6/24 moments=0 但 realtime 有数据 |
| Python 调用禁止 curl | 绕过 VPN SNI 拦截 | 6/23 curl 返回空 |

## 6. CGM 血糖 × 行为 × 状态（v4.8）

| 测试项 | 预期 | 曾失败 |
|--------|------|--------|
| `.hae` 从文件头 LZFSE 解压 | 得到 JSON，非空数据日可解析 | 把 `bvx2` 误当私有 header 后跳过导致解压失败 |
| mmol/L + mg/dL 双单位去重 | 每 5 分钟只保留一个 mmol/L 点 | 未统一单位会令均值/峰值荒谬 |
| 覆盖 <20h | 显示实际覆盖，并明确晚间结论留空 | 用半天数据评价全天 |
| Looki 描述提到食物、标题是通勤/运动 | 不建立餐次窗口 | 说明文字串场造成假餐次 |
| 明确餐食标题 | 输出餐前 30min 中位数、餐后峰及约2h值 | 只用单次 delta 比较餐次 |
| 餐后有明确运动场景 | 只记录 120min 内的已标注活动 | 未记录行为却假定饭后散步 |
| RHR/HRV/睡眠缺一项 | 仅报告血糖形态，不做血糖-状态对应 | 恢复数据缺失仍称状态稳定 |
| 血糖平稳而恢复偏离 | 写“不同步，不能归因于血糖波动” | 单日时间相关被写成因果 |
